## SimpleForm 變體指南（指引）

當你要設計新的 SimpleForm 變體時，請依照本文件操作。目標是在保持一致的使用者行為下，探索不同的視覺與互動風格。

**範圍**
- 在 `src/SimpleForm/` 建立新的元件，行為需與現有變體一致。
- 只能調整樣式、微互動與動態語言。
- 程式結構需清楚，並與現有元件一致。

**開始方式**
- 複製 `src/SimpleForm/StarterVariant.jsx` 並改名為你的變體。
- 更新標題與說明文字以符合主題。
- 更換色彩、形狀與動態 class，保持狀態邏輯不變。

**必須一致的使用者行為（不可變更）**
- 單一文字輸入框，含 placeholder。
- 打字時會觸發短暫的「輸入脈衝」（約 400ms），輸入非空後視為啟用。
- 輸入為空時，主按鈕必須禁用。
- 在啟用狀態點擊按鈕後進入 `working`。
- 點擊後約 180ms 顯示延遲的 spinner。
- 約 3 秒後轉為 `success`。
- 成功狀態顯示確認（`Done` + 勾勾），並在約 1.6 秒後回到 `idle`。
- 在 `working` 與 `success` 期間，輸入框與按鈕需鎖定。

**必須存在的狀態**
- `idle`（預設）。
- `working`（延遲後顯示 spinner）。
- `success`（顯示成功視覺）。

**元素對照表**
| 元素 | 子元素 | 行為 |
| --- | --- | --- |
| 容器 | 標題、說明文字、輸入區、操作區 | 靜態結構，用於承載主題與氛圍。 |
| 標題 | 簡短標籤 | 無狀態變化，用來定義語氣。 |
| 說明文字 | 一行描述 | 無狀態變化，說明意圖與主題。 |
| 輸入欄位 | Placeholder、游標、聚焦外觀 | 非空啟用；打字觸發短暫脈衝；`working`/`success` 時鎖定。 |
| 輸入外框 | 邊框、背景、內光 | 依啟用/輸入/聚焦狀態變化；不改變邏輯。 |
| 主按鈕 | 文字、箭頭、基礎樣式 | 輸入為空禁用；點擊進入 `working`；`working`/`success` 鎖定。 |
| Spinner 層 | Spinner + 可選漸層覆蓋 | `working` 後約 180ms 顯示；其他狀態隱藏。 |
| Success 層 | 閃光、波紋、粒子、勾勾 | `success` 時顯示；約 1.6 秒後重置為 `idle`。 |
| 動態裝飾 | Hover、呼吸、脈衝、光暈 | 可選；不可改變時序與狀態轉換。 |

**可自由設計的部分**
- 色彩、光暈、漸層與邊框。
- 輸入框外型（圓角矩形、膠囊、斜角等）。
- 按鈕外觀（外框、描邊、陰影、玻璃感）。
- 微互動（hover 抬升、呼吸、波紋、光斑）。
- 元件內部的背景處理。
- 文案語氣（標題 + 說明），但意圖需清楚。

**一致性規則**
- 不可新增額外輸入或步驟。
- 不可變更時序、標籤或狀態轉換。
- 不可新增除主要按鈕外的操作。
- 文字需維持：閒置/啟用時 `Start`，成功時 `Done`。
- 保留箭頭與勾勾的提示符號。

**無障礙與 UX 品質**
- 文字需維持足夠對比度。
- 輸入框聚焦時要有可辨識的 focus 樣式。
- 動態不可遮蔽文字或輸入值。

**交付清單**
- 在 `src/SimpleForm/` 新增變體元件檔案。
- 視覺風格需與 IgniteCore、PrismConsole 明顯不同。
- 行為需完全符合上述規則。
- 使用函式型元件與 hooks。
